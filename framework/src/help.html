    • What are modules?

    Well, EVI is a framework, or call it an interface.
    So interface for what? For modules.
    Without modules, EVI is nothing. Modules actually bring the functionality. An IRC- or ICQ-client, a game or any other tool can be written as EVI module. By choosing a bunch of modules, the user customizes the application to fit his needs.

    Where do I get modules?

    You can find and download modules from the EVI project homepage: http://evi.berlios.de.

    You can also load modules directly from the internet. See the next question for information how to do that.

    How to load modules?

    To load modules, choose EVI in the menu at the top and then click Modules.
    In this panel, you have an empty dropdown list at the top and the buttons Add, Browse and Get List (or similar).

    Before you can Add a module, you need to choose one. There are two ways to do this:

    • Go to http://evi.berlios.de, donwload a module JAR and save it to harddisk. Then click Browse in the EVI load-module-panel and choose the downloaded JAR file. Then click the Add button.
    • The other way is to directly let EVI download the module from the internet. Hit the Get List button in the EVI load-module-panel. Moments later, the dropdown box at the left will be filled with a list of URLs. Each URL points to a module JAR file. Choose any module that pleases you and press the Add button.

    After hitting Add, the module is added to the table at the bottom. The module will be loaded in future.

    A short notice about the second way (let EVI download the module): This is nice to quickly test a module. The disadvantage is that the module is downloaded each time EVI starts. Hence, the startup time increases. You might want to download the module's JAR file manually from the internet and then load it from harddisk.

    What means "Auto Start"?

    You might want EVI to start specific modules when the application starts. An example: you might want the IRC- or ICQ-client to be started and connected automatically.

    You can tell EVI to do so. Click EVI in the menubar and choose Auto Start.
    A panel will appear with to tables: The first table lists all loaded modules, the second shows those that will be started automatically.

    You can either rightclick a row in the first table and click Add to Auto Start or simply double click it. The module will appear in the bottom table.

    Some modules can take arguments. These arguments let them know what the should do when they're started. You can pass arguments to it by doing rightclick on a row in the bottom table and then click Edit Arguments. The cursor will jump into the right cell of the selected row and you can type the arguments.
    Such an argument might be the address of the server the module should connect to when it's started or a URL or something like that.
